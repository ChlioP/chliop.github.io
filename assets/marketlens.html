Application Preview

MarketLens is a financial research application that transforms external market data into a structured and responsive dashboard. Users can search stock symbols, review company and quote information, explore historical price movement, save tickers to a local watchlist, revisit recent searches, and compare available stock metrics.

The original application used a basic form-and-tabs layout. I redesigned it into a market intelligence dashboard with stronger information hierarchy, interactive visualizations, responsive layouts, and clearer handling of missing or delayed financial data.

Tools & Technologies

  • Python and Flask for backend routing and API requests
  • HTML5 and CSS3 for dashboard structure and responsive design
  • JavaScript for search interactions, watchlists, and UI state
  • Chart.js for historical stock price visualization
  • LocalStorage for saved watchlists and recent searches
  • External financial data APIs for company, quote, historical, and news data
  • Render for cloud deployment
  • Git and GitHub for version control

Application Highlights

  • Stock Search: Search company data using ticker symbols such as AAPL, MSFT, NVDA, AMD, and TSLA
  • Company Summary: Display company name, ticker, exchange, previous close, update time, and data availability
  • Historical Price Chart: Explore price movement across multiple time ranges
  • Watchlist: Save and reopen ticker symbols using browser LocalStorage
  • Recent Searches: Store recent successful ticker searches for faster navigation
  • Stock Comparison: Compare available metrics across two selected companies
  • Financial News: Display recent company-related articles when supported by the API
  • Responsive Layout: Support desktop, tablet, and mobile screen sizes

Financial Data Quality Handling

Financial APIs may return delayed quotes, incomplete records, missing historical prices, or temporary request failures. The application validates returned values before displaying them to prevent missing data from appearing as legitimate financial values.

  • Missing latest price values display as “Price unavailable” instead of $0.00
  • Missing market index values display a compact unavailable status
  • Missing historical data uses a dedicated chart empty state
  • Invalid ticker symbols return a clear user-facing error message
  • Delayed financial values are labeled so users do not interpret them as real-time trading signals

Technical Challenges

Inconsistent API Responses

Financial endpoints sometimes returned missing or differently formatted values. I added reusable validation and formatting logic before rendering financial metrics.

Chart Layout Stability

The historical chart previously expanded during responsive resizing. I constrained its container height and controlled chart recreation to prevent browser lag and layout growth.

Dashboard Hierarchy

I replaced the original form-and-tabs interface with a two-column research dashboard that prioritizes company data and historical analysis while keeping watchlist tools secondary.
